Food Delivery Time Prediction (外送時間預測專案)
本專案是一個完整端到端的資料科學與機器學習專案，旨在透過外送員個人特質、外送載具、訂單類型以及真實行車距離，精準預測外送服務的所需時間。
專案中特別引入了 OSRM API 來取代傳統的直線距離，以取得更貼近現實道路狀況的實際行車公里數，並利用 XGBoost 迴歸模型 結合網格搜尋 (GridSearchCV) 進行模型的訓練與優化。

專案核心特色
1. 真實行車距離計算：使用 OSRM API 傳入餐廳與外送地點的經緯度，動態計算實際的「道路行駛距離」，大幅提升特徵的業務合理性。
2. 嚴謹的資料清理與防錯：
   a. 自動過濾全球經緯度異常值。
   b. 針對離群值（如評分大於 5 分、外送員年齡不合常理、OSRM 計算失敗或距離大於 100km 等）進行清洗與修正。
3. 完整特徵工程：自動對類別變數（如訂單類型 Type_of_order、載具類型 Type_of_vehicle）進行 One-Hot Encoding。
4. 超參數自動化調優：內建 GridSearchCV 進行交叉驗證，針對 n_estimators, max_depth, learning_rate, subsample, gamma 等核心超參數進行網格搜尋。
5. 豐富的資料視覺化與診斷圖表：
   a.資料探索：Pairplot、相關係數熱力圖、外送特徵與時間的箱線圖。
   b. 模型診斷：Top 5 核心特徵重要性分析圖、殘差分佈直方圖以及模型學習曲線。

使用說明

步驟 1：執行資料預處理與特徵工程
執行 food.py。程式將讀取原始的 Excel 資料集、過濾異常、調用 OSRM API 計算距離，並將切分好的 80% 訓練集與 20% 測試集自動存成 CSV 檔案，同時彈出 EDA 探索性資料分析圖表。
步驟 2：訓練 XGBoost 模型與超參數調優
確保目錄下已生成 X_train.csv 等檔案後，執行 train_xgboost.py。程式會自動啟動多核心（n_jobs=-1）進行網格搜尋，找出最佳模型，並輸出最終的預測評估指標與殘差圖。
